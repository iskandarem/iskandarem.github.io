Setting Up Jekyll on GitHub Pages

Published June 28, 2025

If you want a simple way to create a blog or portfolio with no backend, Jekyll and GitHub Pages make a great combo. Hereâ€™s how I set it up.

1. Create Your GitHub Repository

Create a new repo named exactly:

yourusername.github.io

This tells GitHub to treat it as a personal site and serve it at that URL.

2. Add Jekyll Files

You can add files manually or use a theme. A minimal Jekyll site might look like this:

_config.yml
index.md
_layouts/
  default.html

Or use a theme from github.com/pages-themes.

3. Enable GitHub Pages

In your repo settings, go to Pages and set the source to your main branch. GitHub will build and serve your site at:

https://yourusername.github.io

4. Write a Blog Post

Posts go in a folder named _posts. The filename format is important:

_posts/2025-06-28-my-first-post.md

Add front matter at the top:

---
layout: post
title: "My First Blog Post"
---

Hello world! This is my first blog post powered by Jekyll.

5. You're Done

Push changes to GitHub, and your site updates automatically. No need for builds or deployment scripts.
